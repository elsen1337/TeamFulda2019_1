Infrastruktur-Gedöhns

Anmerkung: hsftp etwas kürzer als das hsfss19tp1; weniger Tipparbeit ;-)
  • Generischer Server: pizarro.uberspace.de; Web-Alias: hsftp.uber.space
  • CentOS7 (RHEL Fork), Apache 2.4 Webserver mit nGinx HTTPS Frontent mit HTTP/2 und Let's Encrypt
  • 10GB Storage; bis zu 1.5 GB RAM

MySQL bzw. Maria DB 10.3 (MySQL - Fork)

User: hsftp; PW: per Mail
  • phpMyAdmin
  • Adminer
Jeder der mit dem Datenbankdesign loslegen will, soll und darf das gerne tun. Einzige Voraussetzung: Entwurfstabellen bitte mit Namespräfix versehen; also z. B. für eine Zimmertabelle name_rooms; die Produktionstabellen haben dann kein Namens-Präfix mehr. Bei n SQL Entitäten existieren dann also max. (5+1)n = 6n Tabellen, sofern jeder einen Vorschlagsentwurf macht. Idealerweise haben die Tabellen auch ein Projektpräfix also msctprb (für Master-TeamProjekt-Room-Booking), sodass msctprb_rooms die Produktionstabelle und name_rooms die Entwurfstabelle ist.

GIT/SSH

  • Login über eure Key-Pairs - Push geht teilweise sehr langsam; nochmal auf Fehlerursache gehen. (Gab da einen Hinweis im Wiki https://wiki.uberspace.de/development:git...) bzw. ElGamal - EllepticCurve sind sehr rechenintensiv; vllt. wenn es stört auf RSA zurückgehen, was die Erstellung (ssh-keygen -t [...]) eines neuen Schlüsselpaars zur Folge hätte.
  • Spätestens im Projekt-Branch wird mit mindestens einer Nutzerbranche gearbeitet. Somit kann zwischen Team (master - Branch) und Mitglieder Anteil unterschieden werden. Anmerkung: Dies könnte man auch alleine schon über den Nutzer; aus organiatorischer Sicht ist es naiv alles sofort in die master - Branch zu pushen. Standard - Branch für jeden sei vorname; ob weitere Branches (Namensschema: vorname_feature) erstellt werden ist jedem selbst überlassen.
Für das eigentliche Projekt habe ich noch kein Repository angelegt; erstmal auf Testing herumspielen. Wie ihr an der URL seht wird das GIT-Bare-Repository automatisch (!) in ein Arbeitsvereichnis geschrieben, das unter https://hsftp.uber.space/reponame erreichbar ist. Dateien erhalten chmod 0604, Verzeichnisse 0705.

Lokale Umgebung

  • Weil ein Ausprobieren und flüssiges Entwickeln sinnvoll ist, sollte jeder eine eine eigene Webserver-PHP-MySQL Umgebung aufsetzen. Von der eigenen Branch kann lokal auf die master - Branch gemert werden. Ein Tutorial findet sich u.a. hier: Branching/Merging oder hier Rebase VS Merge
  • Für Windows kann ich die Umgebung uwAMPP empfehlen; auf Linux empfieht sich ein Apache - mod_proxy_fcgi (mit Sockets) - PHP-FPM mit MySQL/MariaDB Setup. Tutorial für Debian 9 (Ignoriere den Drupal-Krempel)

Errschtte Schritte...
